Java序列化

  1. Java序列化是指把Java对象转换为字节序列的过程，而java反系列化是指把自己饿序列恢复为Java对象的过程；

  2. 序列化主要用处是就是在传递和保存对象的时候，保证对象的保证行和可传递性；序列化是把对象转换有序字节流，以便在网络上传输或者保存在本地文件中。核心作用是对象状态的保存与重建；

  3. 反序列化：客户端从文件中或者网络上获得序列化后的对象字节流，根据字节流中所保存的对象状态及描述信息，通过返序列化重建对象；

为什么要序列化和反序列化？

为什么要序列化？

  1. 对象序列化可以实现分布式对象；

主要应用例如：RMI要利用对象序列化运行远程主机上的服务，就像本地机上运行对象时一样；

  1. java对象序列化不仅仅保留对象的数据，而且递归保存对象引用的每个对象的数据；

  2. 序列化可以将内存中的类写入文件或数据库中；

  3. 对象、文件、数据，有许多不同的格式，很难统一传输和保存；

什么是java bean

  1. bean是对象；

  2. 由Spring中一个叫loC的东西管理；

  3. 我们的应用程序由一个一个bean构成；

Bean的作用域

Bean在Spring中的某种行为模式：

Bean对象是默认的单例作用域，如果有任何地方有任何修改了这个单例的对象，那么其他类再调用就会得到一个修改后的值；

作用域分类

  1. singleton：单例作用域；

  2. prototype：原型作用域；

  3. request：请求作用域；

  4. session： 会话作用域；

  5. application：全局作用域；

后三种作用域，只适用于Spring MVC框架；

bean规范

  1. 所有属性为private；

  2. 提供默认构造方法；

  3. 提供setter和getter；

  4. 实现serializable接口；

window 窗口

  • 把无限流中切割为有限流的方式，它会将流数据分发到有限大小的桶（bucket）中进行分析；

  1. 时间窗口

    • 滚动时间窗口

    • 滑动时间窗口

    • 会话窗口

  2. 计数窗口

    • 滚动计数窗口

    • 滑动计数窗口

滚动窗口（Tumbling windows）

  • 将数据依据固定的窗口长度对数据进行切分；

  • 时间对齐，窗口长度固定，没有重叠；

滑动窗口（Sliding Windows）

  • 滑动窗口是固定窗口的更广泛的一种形式，滑动窗口由固定的窗口长度和滑动间隔组成

  • 窗口长度固定，可以重叠；

会话窗口（Session Windows）

  • 由一系列事件组合一个指定时间长度的timeout间隙组成，也就是一段时间没有接收到新数据就会生成新的窗口；

  • 特点：时间无对齐

Window API

窗口函数（windows function）

  • window function定义了要对窗口手机的数据做计算操作

  • 可以分为两类：

    • 增量聚合函数

      • 每条数据到来就计算，保存一个简单状态

      • ReduceFunction AggregateFunction

    • 全窗口函数

      • 先把窗口所有数据收集起来，等到计算的时候会遍历所有数据

      • ProcessWindowsFunction，WindowFunction
